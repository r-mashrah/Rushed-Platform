# مخططات تطبيق إدارة الفصول والاختبارات

## 📋 نظرة عامة

هذا المجلد يحتوي على مخططات PlantUML شاملة لتطبيق إدارة الفصول والاختبارات. تغطي المخططات جميع جوانب المشروع من حالات الاستخدام إلى تدفق البيانات والحالات الداخلية.

---

## 📊 قائمة المخططات

### 1️⃣ مخطط حالات الاستخدام (Use Case Diagram)
**الملف:** `01_use_case_diagram.puml`

#### الوصف:
يوضح جميع حالات الاستخدام الممكنة في التطبيق من قبل جميع الأطراف الفاعلة (المعلم، الطالب، نظام الذكاء الاصطناعي).

#### الأطراف الفاعلة:
- **المعلم**: ينشئ الفصول والاختبارات والأسئلة
- **الطالب**: يحل الاختبارات ويرى النتائج
- **نظام الذكاء الاصطناعي**: يولد أسئلة ذكية

#### حالات الاستخدام الرئيسية:
| الرقم | الحالة | الوصف |
|------|--------|-------|
| UC01 | التسجيل والدخول | المصادقة والتحقق من البيانات |
| UC03 | إدارة الفصول | إنشاء وتعديل الفصول |
| UC05 | إنشاء اختبار | تكوين وإعداد الاختبارات |
| UC06 | بناء بنك أسئلة | إضافة وتنظيم الأسئلة |
| UC07 | توليد أسئلة ذكية | استخدام الذكاء الاصطناعي |
| UC10 | الإجابة على الأسئلة | حل الاختبار من قبل الطالب |
| UC12 | تحليل الأداء | عرض إحصائيات الأداء |
| UC13 | تقرير الفجوات | تحديد الفجوات التعليمية |

### 2️⃣ مخطط الفئات (Class Diagram)
**الملف:** `02_class_diagram.puml`

#### الوصف:
يوضح البنية البرمجية للتطبيق بما يشمل:
- النماذج (Models)
- الخدمات (Services)
- المستودعات (Repositories)
- المتحكمات (Controllers)
- الواجهات (Views)

#### الفئات الرئيسية:

##### 📦 Models
```
TeacherModel      - بيانات المعلم
StudentModel      - بيانات الطالب
ClassModel        - بيانات الفصل
QuestionModel     - بيانات السؤال
QuestionOption    - خيارات السؤال
NotificationModel - الإخطارات
```

##### 🔧 Services
```
AuthService                    - المصادقة والتسجيل
AiService                      - توليد الأسئلة الذكية
StorageService                 - تخزين البيانات المحلية
QuestionAnalysisService        - تحليل جودة الأسئلة
CurriculumGapAnalysisService   - تحليل الفجوات التعليمية
```

##### 📚 Repositories
```
QuestionRepository        - إدارة الأسئلة من DB
ClassesRepository         - إدارة الفصول من DB
NotificationsRepository   - إدارة الإشعارات من DB
AttendanceRepository      - إدارة الحضور من DB
PendingContentRepository  - إدارة المحتوى المعلق من DB
```

##### 🎮 Controllers
```
LoginController           - منطق تسجيل الدخول
DashboardController       - منطق الصفحة الرئيسية
QuizBuilderController     - منطق بناء الاختبار
StudentDetailController   - منطق التفاصيل
```

#### العمليات الرئيسية:

| الرقم | العملية | الوصف |
|------|---------|-------|
| 1.0 | نظام المصادقة والتسجيل | التحقق من بيانات المستخدم |
| 2.0 | إدارة الفصول والطلاب | استرجاع وإدارة بيانات الفصول |
| 3.0 | إنشاء وإدارة الأسئلة | إضافة تعديل حذف الأسئلة |
| 4.0 | إنشاء الاختبارات | دمج الأسئلة في اختبار |
| 5.0 | تسليم الاختبارات | إرسال الاختبار للطلاب |
| 6.0 | تقييم الإجابات | حساب النتائج |
| 7.0 | تحليل الأداء | حساب الإحصائيات |
| 8.0 | إنشاء التقارير | توليد التقارير |
| 9.0 | إدارة الإشعارات | إرسال الإشعارات |

#### مستارات البيانات:

| الرقم | المستودع | الوصف |
|------|----------|-------|

| D8 | الجودة والتحليلات | مؤشرات جودة الأسئلة |
| D9 | الحضور | سجلات الحضور |

### 4️⃣ مخطط تسلسل الأحداث (Sequence Diagram)
**الملف:** `04_sequence_diagram.puml`

#### الوصف:
يوضح ترتيب وتسلسل التفاعلات بين المكونات المختلفة للعمليات الرئيسية.

#### السيناريوهات المغطاة:

##### 🔐 السيناريو 1: عملية التسجيل والدخول
```
المستخدم 
  → LoginView 
    → LoginController 
      → AuthService 
        → Supabase API
```
- التحقق من البيانات
- التوثيق عبر Supabase
- حفظ الجلسة في التخزين المحلي

##### 📝 السيناريو 2: إنشاء واختبار
```
المعلم
  → QuizBuilderView
    → QuizBuilderController
      → AiService
        → Gemini API (توليد الأسئلة)
          → QuestionRepository
            → Supabase (حفظ الأسئلة والاختبار)
```
- اختيار المادة والفصل
- توليد أسئلة ذكية
- اختيار الأسئلة وترتيبها
- حفظ الاختبار

##### ✏️ السيناريو 3: الطالب يحل الاختبار
```
الطالب
  → ExamView
    → ExamController
      → ExamRepository
        → Supabase (حفظ الإجابات والنتائج)
```
- استرجاع الأسئلة
- الإجابة على كل سؤال
- حساب النتيجة
- حفظ الإجابات والإحصائيات

##### 📊 السيناريو 4: تحليل الأداء والفجوات
```
المعلم
  → AnalysisView
    → AnalysisController
      → QuestionAnalysisService
      → CurriculumGapAnalysisService
        → ClassesRepository
          → Supabase (استرجاع البيانات)
```
- تحليل جودة الأسئلة
- تحديد الفجوات المنهجية
- حساب مؤشرات الأداء

### 5️⃣ مخطط الحالات الداخلية (State Diagram)
**الملف:** `05_state_diagram.puml`

#### الوصف:
يوضح جميع الحالات المختلفة التي يمكن أن تكون فيها الكائنات المختلفة في النظام وكيفية الانتقال بينها.

#### الحالات الرئيسية:

##### 🎓 حالات الطالب في الاختبار:
```
NotEnrolled 
  → Assigned 
    → NotStarted 
      → InProgress 
        → [TimeExpired | Submitted] 
          → Graded
```

##### ❓ حالات الأسئلة:
```
Draft 
  → PendingReview 
    → [Approved | Rejected]
      → Active (مع تطور الإحصائيات)
        → Archived
```

**مؤشرات جودة السؤال:**
- **Discrimination Index** > 0.7: ممتاز
- **Discrimination Index** 0.5-0.7: جيد
- **Discrimination Index** 0.3-0.5: مقبول
- **Discrimination Index** < 0.3: ضعيف ويحتاج مراجعة

##### 📋 حالات الاختبار:
```
Draft 
  → InDesign 
    → ReadyToPublish 
      → Published 
        → Active 
          → Closed 
            → Graded
```

##### 👨‍🏫 حالات نشاط المعلم:
من تسجيل الدخول إلى مراجعة النتائج والتقارير

##### 👨‍🎓 حالات نشاط الطالب:
من تسجيل الدخول إلى الإجابة على الاختبارات ومراجعة النتائج

## 🛠️ كيفية استخدام المخططات

### عرض المخططات:

#### الخيار 1: عبر موقع PlantUML Online
```
1. اذهب إلى https://www.plantuml.com/plantuml/uml/
2. انسخ محتوى أي ملف .puml
3. الصقه في المحرر
4. سيتم عرض المخطط تلقائياً
```

#### الخيار 2: استخدام VS Code Extension
```
1. ثبت الإضافة: PlantUML
   Publisher: jebbs
   
2. افتح أي ملف .puml
3. اضغط Ctrl+Shift+P واكتب "PlantUML: Preview"
4. سيتم عرض المخطط في نافذة جانبية
```

#### الخيار 3: تصدير إلى صور
```
plantuml diagrams/*.puml -o output_folder/ -tpng
```
